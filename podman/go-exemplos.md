# Exemplos Go com Podman

Este arquivo concentra os exemplos em Go que fazem sentido para o curso usando Podman.

A ideia aqui e usar Go como continuação natural do laboratório:

- primeiro subir o Oracle;
- depois validar a conexão;
- por fim conectar uma aplicação real.

Os exemplos abaixo usam os projetos reais do workspace:

- `repo/go.server`
- `repo/go.oracle/v1`

## 1. Exemplo Basico: `repo/go.server`

Esse projeto sobe uma API HTTP simples em Go na porta `8080`.

Endpoint:

```txt
POST /api/v1/user
```

### Rodar com `podman run` e `go run`

```bash
cd repo/go.server

podman run --rm -it \
  -p 8080:8080 \
  -v "$PWD":/app:Z \
  -w /app \
  golang:1.25 \
  go run main.go
```

Teste:

```bash
curl -i -X POST http://localhost:8080/api/v1/user \
  -H 'Content-Type: application/json' \
  -d '{"nome":"Jeff","stack":"Go + Podman"}'
```

### Buildar imagem local com `Containerfile`

```bash
cd repo/go.server
podman build -t go-server-run -f Containerfile .
podman run --rm -it -p 8080:8080 go-server-run
```

### Outras variantes do projeto

O diretorio tem outros exemplos de build:

```txt
repo/go.server/Containerfile.build
repo/go.server/Containerfile.alpine
repo/go.server/Containerfile.distroless
repo/go.server/Containerfile.scratch
```

Esses arquivos sao bons para explicar diferenca entre:

- imagem maior para desenvolvimento;
- imagem enxuta para execucao;
- imagem distroless;
- imagem scratch.

## 2. Exemplo Go + Oracle: `repo/go.oracle/v1`

Esse projeto sobe uma API em Go conectando no Oracle Free Full 23ai.

Endpoint base:

```txt
/api/v1/llm
```

Endpoints:

```txt
GET    /health
GET    /api/v1/llm
GET    /api/v1/llm?id=1
POST   /api/v1/llm
PUT    /api/v1/llm?id=1
DELETE /api/v1/llm?id=1
```

### Subir Oracle antes

O Go depende do Oracle rodando em:

```txt
localhost:1522/FREEPDB1
```

O fluxo Oracle completo esta em:

```txt
podman/oracle-exemplos.md
```

### Rodar a API Go localmente

```bash
cd repo/go.oracle/v1
go run .
```

Variaveis padrao:

```txt
ORACLE_DSN=oracle://system:OraclePwd123@localhost:1522/FREEPDB1
HTTP_ADDR=:8080
```

Se quiser explicitar:

```bash
cd repo/go.oracle/v1
ORACLE_DSN='oracle://system:OraclePwd123@localhost:1522/FREEPDB1' \
HTTP_ADDR=':8080' \
go run .
```

### Criar a tabela no Oracle

Use o CloudBeaver e execute:

```sql
CREATE TABLE modelos_llms (
  id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nome VARCHAR2(120) NOT NULL,
  input_token_limit NUMBER NOT NULL,
  output_token_limit NUMBER NOT NULL,
  provider VARCHAR2(80) DEFAULT 'openai' NOT NULL,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL
);
```

### Healthcheck

```bash
curl -s http://localhost:8080/health
```

### Criar um registro

```bash
curl -s -X POST http://localhost:8080/api/v1/llm \
  -H 'Content-Type: application/json' \
  -d '{
    "nome": "gpt-5.4",
    "input_token_limit": 200000,
    "output_token_limit": 128000,
    "provider": "openai"
  }'
```

### Listar registros

```bash
curl -s http://localhost:8080/api/v1/llm
```

### Buscar por ID

```bash
curl -s 'http://localhost:8080/api/v1/llm?id=1'
```

### Atualizar

```bash
curl -s -X PUT 'http://localhost:8080/api/v1/llm?id=1' \
  -H 'Content-Type: application/json' \
  -d '{
    "nome": "gpt-5.4-mini",
    "input_token_limit": 128000,
    "output_token_limit": 64000,
    "provider": "openai"
  }'
```

### Remover

```bash
curl -s -X DELETE 'http://localhost:8080/api/v1/llm?id=1'
```

## 3. Testes


## 4. Quando usar cada exemplo

- `repo/go.server`: primeiro contato com Podman + Go + HTTP simples.
- `repo/go.oracle/v1`: segundo passo, agora com Oracle real e CRUD.
- `podman run` com `golang`: bom para demonstracao e desenvolvimento rapido.
- `podman build`: bom para empacotar uma imagem local.

## 5. Sequência recomendada

1. subir o Oracle pelo guia `podman/oracle-exemplos.md`;
2. validar a conexão no CloudBeaver;
3. criar a tabela `modelos_llms`;
4. rodar `repo/go.oracle/v1`;
5. testar o CRUD com `curl`.
