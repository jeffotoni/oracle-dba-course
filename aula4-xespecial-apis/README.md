# Aula 4 Especial - APIs com Go + Oracle

> Aula prática para rodar, testar e entender uma API HTTP em Go conectada ao Oracle usando `repo/go.oracle/v1`.

## Objetivo da aula

Nesta aula, o foco é mostrar uma API real conectando aplicação e banco de dados.

O objetivo é:v

- subir uma versão Oracle com `Podman`;
- criar a tabela do laboratório;
- rodar a API localmente com `go run`;
- compilar com `go build`;
- testar `GET`, `POST`, `PUT` e `DELETE`;
- mostrar a ponte entre aplicação em Go e dados no Oracle;
- preparar terreno para a aula seguinte com `ORDS`.

## Projeto usado na aula

- código-fonte: `repo/go.oracle/v1/main.go`
- guia técnico do projeto: `repo/go.oracle/v1/README.md`
- rota principal: `/api/v1/llm`

## Linha de raciocínio da aula


## O que a API faz

A API usa:

- Go;
- `net/http`;
- `database/sql`;
- driver Oracle `go-ora`;
- Oracle Free em container.

Ela expõe:

```txt
GET    /health
GET    /api/v1/llm
GET    /api/v1/llm?id=1
POST   /api/v1/llm
PUT    /api/v1/llm?id=1
DELETE /api/v1/llm?id=1
```

### O que ela persiste

A tabela usada é:

```txt
modelos_llms
```

Campos principais:

- `id`
- `nome`
- `input_token_limit`
- `output_token_limit`
- `provider`
- `created_at`

## Quick start

## 1. Escolher uma versão Oracle para a aula

O ideal é escolher **uma** versão para toda a aula e manter o mesmo padrão de porta e conexão.

As mais adequadas aqui são:

- `repo/oracle/versoes/free-full-23ai`
- `repo/oracle/versoes/free-lite-23ai`
- `repo/oracle/versoes/oracle-12c`

### Recomendação para a aula

Use:

- `free-full-23ai`

Porque ela já está mais alinhada com os exemplos do curso.

## 2. Subir Oracle com Podman

### Opção recomendada: `free-full-23ai`

```bash
cd repo/oracle/versoes/free-full-23ai/script
./up.sh
```

### Se quiser usar a versão lite

```bash
cd repo/oracle/versoes/free-lite-23ai/script
./up.sh
```

### Se quiser usar a versão 12c

```bash
cd repo/oracle/versoes/oracle-12c/script
./up.sh
```

## 3. Validar Oracle

Exemplo para `free-full-23ai`:

```txt
Host: localhost
Port: 1522
Service Name: FREEPDB1
User: system
Password: OraclePwd123
```

## 4. Criar a tabela da API

Executar em `Oracle SQL Developer`, `CloudBeaver`, `DBeaver` ou equivalente:

```sql
CREATE TABLE modelos_llms (
  id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nome VARCHAR2(120) NOT NULL,
  input_token_limit NUMBER NOT NULL,
  output_token_limit NUMBER NOT NULL,
  provider VARCHAR2(80) DEFAULT 'openai' NOT NULL,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL
);
```

## 5. Rodar a API localmente

```bash
cd repo/go.oracle/v1
go run .
```

## 6. Compilar

```bash
cd repo/go.oracle/v1
go build -o go-oracle-api .
```

## 7. Rodar o binário compilado

```bash
cd repo/go.oracle/v1
./go-oracle-api
```

## Endpoints que valem mostrar ao vivo

### Healthcheck

```bash
curl -i http://localhost:8080/health
```

### GET de listagem

```bash
curl -i http://localhost:8080/api/v1/llm
```

### GET por ID

```bash
curl -i 'http://localhost:8080/api/v1/llm?id=1'
```

### GET por ID inexistente

```bash
curl -i 'http://localhost:8080/api/v1/llm?id=999'
```

### POST criando um modelo

```bash
curl -i \
  -X POST http://localhost:8080/api/v1/llm \
  -H 'Content-Type: application/json' \
  -d '{
    "nome": "gpt-5.4",
    "input_token_limit": 200000,
    "output_token_limit": 128000,
    "provider": "openai"
  }'
```

### POST criando outro modelo

```bash
curl -i \
  -X POST http://localhost:8080/api/v1/llm \
  -H 'Content-Type: application/json' \
  -d '{
    "nome": "claude-3.7-sonnet",
    "input_token_limit": 200000,
    "output_token_limit": 64000,
    "provider": "anthropic"
  }'
```

### PUT atualizando um modelo

```bash
curl -i \
  -X PUT 'http://localhost:8080/api/v1/llm?id=1' \
  -H 'Content-Type: application/json' \
  -d '{
    "nome": "gpt-5.4-mini",
    "input_token_limit": 128000,
    "output_token_limit": 64000,
    "provider": "openai"
  }'
```

### DELETE removendo um modelo

```bash
curl -i \
  -X DELETE 'http://localhost:8080/api/v1/llm?id=1'
```

## O que vale observar na aula

- API própria tem rota, handler e SQL definidos pela aplicação;
- a aplicação depende do Oracle estar no ar e com tabela criada;
- `GET`, `POST`, `PUT` e `DELETE` aparecem de forma concreta;
- a resposta HTTP ajuda a explicar fluxo de CRUD;
- o log da aplicação ajuda a mostrar conexão e operação.

## Comparação didática com ORDS

Esta aula prepara a seguinte comparação:

### API em Go

- a aplicação define rota;
- a aplicação define handler;
- a aplicação escreve SQL;
- a aplicação controla a resposta HTTP.

### API com ORDS

- o Oracle expõe objetos e dados por HTTP/JSON;
- a base de dados participa diretamente da exposição da API;
- a lógica fica mais próxima do banco.

## Mensagem final da aula

```txt
Go + Oracle mostra a API controlada pela aplicação
ORDS mostra a API controlada pelo ecossistema Oracle
```
