# Grupo 3 - Busca Vetorial Simples com Oracle

## Tema

Construir uma ferramenta simples de busca semântica usando Oracle como banco vetorial.

## Objetivo

O grupo deve montar uma solução pequena que siga esta linha:

```txt
Texto -> Embedding local -> Vetor no Oracle -> Similaridade -> resultado
```

Se quiserem evoluir um pouco mais:

```txt
Texto -> Embedding local -> Vetor no Oracle -> Similaridade -> contexto -> LLM
```

## Ideia do projeto

A proposta é criar uma **base de conhecimento simples**.

Exemplos possíveis:

- FAQ interno;
- catálogo de documentos curtos;
- base de políticas;
- base de produtos;
- base de prompts;
- base de tickets ou respostas frequentes.

O grupo grava:

- `titulo`
- `texto`
- `embedding`

Depois faz uma busca por similaridade a partir de uma pergunta.

## O que o trabalho deve demonstrar

- o que é embedding;
- como gerar embedding localmente;
- como armazenar vetor no Oracle;
- como consultar por similaridade;
- como mostrar os resultados mais próximos.

## Escopo mínimo

### 1. Banco Oracle

Criar uma tabela vetorial simples.

Exemplo:

```sql
CREATE TABLE kb_docs (
  id         NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  titulo     VARCHAR2(200) NOT NULL,
  texto      CLOB NOT NULL,
  embedding  VECTOR(384, FLOAT32) NOT NULL,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL
);
```

> Observação: `384` é um tamanho compatível com alguns modelos leves. Se o grupo usar outro modelo, a dimensão deve seguir o modelo escolhido.
>
> Regra prática: a dimensão da coluna `VECTOR(...)`, o array usado no `TO_VECTOR(...)` e o embedding gerado precisam ser exatamente compatíveis entre si.

## 2. Geração de embedding

O grupo pode escolher uma das abordagens abaixo.

### Opção A - Ollama local

Instalação:

```bash
brew install ollama
ollama pull nomic-embed-text
```

Exemplo de chamada:

```bash
curl http://localhost:11434/api/embeddings -d '{
  "model": "nomic-embed-text",
  "prompt": "Go é uma linguagem eficiente para APIs"
}'
```

### Opção B - Python com `sentence-transformers`

Instalação:

```bash
pip install sentence-transformers
```

Exemplo:

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
embedding = model.encode("Go é incrível")

print(len(embedding))
print(embedding[:10])
```

## 3. Insert no Oracle

Depois de gerar o embedding, o grupo deve gravar o vetor no Oracle.

Exemplo didático:

```sql
INSERT INTO kb_docs (titulo, texto, embedding)
VALUES (
  'Reset de senha',
  'Para resetar a senha, acesse o portal e clique em esqueci minha senha.',
  TO_VECTOR('[0.12, -0.03, 0.44, 0.08, -0.21, 0.30, 0.09, -0.11]', 8, FLOAT32)
);
```

> Observação: esse exemplo está com 8 dimensões só para visualização. No projeto real, o array deve ter exatamente a mesma dimensão do modelo utilizado.

## 4. Consulta por similaridade

Exemplo:

```sql
SELECT id,
       titulo,
       VECTOR_DISTANCE(
         embedding,
         TO_VECTOR('[0.11, -0.02, 0.41, 0.07, -0.19, 0.29, 0.10, -0.10]', 8, FLOAT32),
         COSINE
       ) AS distancia
FROM kb_docs
ORDER BY VECTOR_DISTANCE(
           embedding,
           TO_VECTOR('[0.11, -0.02, 0.41, 0.07, -0.19, 0.29, 0.10, -0.10]', 8, FLOAT32),
           COSINE
         )
FETCH FIRST 5 ROWS ONLY;
```

Leitura prática:

- quanto menor a distância, mais próximo semanticamente;
- o `TOP 3` ou `TOP 5` já resolve bem para apresentação.

## Fluxo sugerido do sistema

### Fluxo 1 - Simples

1. cadastrar documentos;
2. gerar embedding local;
3. gravar no Oracle;
4. digitar uma pergunta;
5. gerar embedding da pergunta;
6. consultar os documentos mais próximos;
7. mostrar os resultados.

### Fluxo 2 - Um pouco mais rico

1. cadastrar documentos;
2. gerar embedding local;
3. armazenar no Oracle;
4. buscar por similaridade;
5. devolver top 5;
6. opcionalmente enviar os resultados para um LLM resumir.
