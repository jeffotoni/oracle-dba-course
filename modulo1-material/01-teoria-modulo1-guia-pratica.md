## 2. Cenário 1 - JSON (criar, inserir e consultar)

### 2.1. Criar tabela com coluna JSON

```sql
CREATE TABLE mod1_json_docs (
  id         NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  doc        CLOB CONSTRAINT mod1_json_docs_ck CHECK (doc IS JSON),
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL
);
```

### 2.2. Inserir JSON de exemplo

```sql
INSERT INTO mod1_json_docs (doc) VALUES (
  q'~{
    "pedido_id": 1001,
    "cliente": {"nome": "Ana", "cidade": "Sao Paulo"},
    "itens": [
      {"sku": "A1", "qtd": 2, "preco": 10.5},
      {"sku": "B2", "qtd": 1, "preco": 20.0}
    ],
    "status": "NOVO"
  }~'
);

INSERT INTO mod1_json_docs (doc) VALUES (
  q'~{
    "pedido_id": 1002,
    "cliente": {"nome": "Bruno", "cidade": "Rio"},
    "itens": [
      {"sku": "C3", "qtd": 3, "preco": 7.0}
    ],
    "status": "NOVO"
  }~'
);

COMMIT;
```

### 2.3. Consultas JSON básicas

Extrair campo simples:

```sql
SELECT id,
       JSON_VALUE(doc, '$.pedido_id' RETURNING NUMBER) AS pedido_id,
       JSON_VALUE(doc, '$.cliente.nome') AS cliente_nome,
       JSON_VALUE(doc, '$.status') AS status
FROM mod1_json_docs
ORDER BY id;
```

Buscar apenas pedidos de um cliente:

```sql
SELECT id
FROM mod1_json_docs
WHERE JSON_VALUE(doc, '$.cliente.nome') = 'Ana';
```

Explodir array de itens com `JSON_TABLE`:

```sql
SELECT d.id,
       JSON_VALUE(d.doc, '$.pedido_id' RETURNING NUMBER) AS pedido_id,
       jt.sku,
       jt.qtd,
       jt.preco
FROM mod1_json_docs d,
     JSON_TABLE(
       d.doc,
       '$.itens[*]'
       COLUMNS (
         sku   VARCHAR2(30) PATH '$.sku',
         qtd   NUMBER       PATH '$.qtd',
         preco NUMBER       PATH '$.preco'
       )
     ) jt
ORDER BY d.id, jt.sku;
```

## 3. Cenário 2 - Fila simples por tabela (didática)

### 3.1. Criar tabela de fila

```sql
CREATE TABLE mod1_event_queue (
  id           NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  event_type   VARCHAR2(60) NOT NULL,
  payload      CLOB CONSTRAINT mod1_event_queue_payload_ck CHECK (payload IS JSON),
  status       VARCHAR2(20) DEFAULT 'PENDING' NOT NULL,
  created_at   TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  processed_at TIMESTAMP
);

CREATE INDEX mod1_event_queue_idx_status_created
  ON mod1_event_queue (status, created_at);
```

### 3.2. Enfileirar eventos (producer)

```sql
INSERT INTO mod1_event_queue (event_type, payload) VALUES (
  'PEDIDO_CRIADO',
  q'~{"pedido_id":1001,"origem":"web"}~'
);

INSERT INTO mod1_event_queue (event_type, payload) VALUES (
  'PEDIDO_CRIADO',
  q'~{"pedido_id":1002,"origem":"app"}~'
);

COMMIT;
```

### 3.3. Consultar fila pendente

```sql
SELECT id, event_type, status, created_at
FROM mod1_event_queue
WHERE status = 'PENDING'
ORDER BY created_at;
```

### 3.4. Consumir 1 evento (consumer)

Passo 1 - reservar o próximo evento (`SKIP LOCKED` evita colisão entre consumidores):

```sql
SELECT id, event_type, payload
FROM mod1_event_queue
WHERE status = 'PENDING'
ORDER BY created_at
FETCH FIRST 1 ROW ONLY
FOR UPDATE SKIP LOCKED;
```

Passo 2 - marcar como processado:

```sql
UPDATE mod1_event_queue
SET status = 'PROCESSED',
    processed_at = SYSTIMESTAMP
WHERE id = :id_consumido;

COMMIT;
```

### 3.5. Verificar processados

```sql
SELECT id, event_type, status, created_at, processed_at
FROM mod1_event_queue
ORDER BY id;
```

## 4. Procedure opcional para consumo automático

```sql
CREATE OR REPLACE PROCEDURE mod1_consume_next_event AS
  v_id mod1_event_queue.id%TYPE;
BEGIN
  SELECT id
    INTO v_id
    FROM mod1_event_queue
   WHERE status = 'PENDING'
   ORDER BY created_at
   FETCH FIRST 1 ROW ONLY
   FOR UPDATE SKIP LOCKED;

  UPDATE mod1_event_queue
     SET status = 'PROCESSED',
         processed_at = SYSTIMESTAMP
   WHERE id = v_id;

  COMMIT;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    NULL;
END;
/
```

Execução:

```sql
BEGIN
  mod1_consume_next_event;
END;
/
```

## 5. Teste opcional em Go (simples)

Usar somente se quiser validar integração externa de forma mínima.

### 5.1. Preparar projeto

```bash
mkdir -p go-mod1-pratica
cd go-mod1-pratica
go mod init go-mod1-pratica
go get github.com/sijms/go-ora/v2
```

### 5.2. Arquivo `main.go`

```go
package main

import (
	"context"
	"database/sql"
	"fmt"
	"log"
	"time"

	_ "github.com/sijms/go-ora/v2"
)

func main() {
	dsn := "oracle://system:Senha123@localhost:1521/FREEPDB1"
	db, err := sql.Open("oracle", dsn)
	if err != nil {
		log.Fatal(err)
	}
	defer db.Close()

	ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
	defer cancel()

	var pending int
	err = db.QueryRowContext(ctx, `
		SELECT COUNT(*)
		FROM mod1_event_queue
		WHERE status = 'PENDING'
	`).Scan(&pending)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("Pendentes na fila: %d\n", pending)

	_, err = db.ExecContext(ctx, `
		INSERT INTO mod1_event_queue (event_type, payload)
		VALUES ('PEDIDO_CRIADO', '{"pedido_id":2001,"origem":"go"}')
	`)
	if err != nil {
		log.Fatal(err)
	}

	_, err = db.ExecContext(ctx, "COMMIT")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println("Evento inserido com sucesso via Go")
}
```

### 5.3. Executar

```bash
go run main.go
```

## 8. Curiosidade - "escutar" evento no Oracle (estilo notification)

No Oracle, o equivalente mais próximo de `LISTEN/NOTIFY` do PostgreSQL é usar **Oracle AQ** com `DEQUEUE` bloqueante.

Resumo prático:

- PostgreSQL: push nativo com `NOTIFY`.
- Oracle: padrão recomendado para notificação com payload é **AQ**.
- comportamento de "escuta": consumer faz `DEQUEUE` com `WAIT` (bloqueia até chegar mensagem ou timeout).

### 8.1. Ideia de fluxo

1. producer faz `ENQUEUE` na fila AQ;
2. consumer (Go) chama rotina de `DEQUEUE` com espera;
3. quando a mensagem chega, o banco retorna o payload.

### 8.2. Exemplo de procedure de consumo bloqueante (didático)

> Observação: exemplo conceitual para laboratório. Em ambiente real, ajustar tipo de payload e tratamento de erro.

```sql
CREATE OR REPLACE PROCEDURE mod1_aq_dequeue_wait(
  p_queue_name IN VARCHAR2,
  p_payload    OUT VARCHAR2
) AS
  v_deqopt  DBMS_AQ.dequeue_options_t;
  v_msgprop DBMS_AQ.message_properties_t;
  v_msgid   RAW(16);
  v_payload VARCHAR2(32767);
BEGIN
  v_deqopt.wait := 30; -- espera ate 30s por mensagem

  DBMS_AQ.DEQUEUE(
    queue_name         => p_queue_name,
    dequeue_options    => v_deqopt,
    message_properties => v_msgprop,
    payload            => v_payload,
    msgid              => v_msgid
  );

  p_payload := v_payload;
END;
/
```

### 8.3. Exemplo Go (loop de escuta simples)

```go
for {
	var payload string

	_, err := db.ExecContext(ctx, `BEGIN mod1_aq_dequeue_wait(:1, :2); END;`, "fila_eventos", sql.Out{Dest: &payload})
	if err != nil {
		// timeout sem mensagem ou erro de conexao; registrar e continuar
		log.Println("aguardando evento:", err)
		continue
	}

	fmt.Println("evento recebido:", payload)
	// processar evento
}
```

### 8.4. Quando usar esse padrão

- quando for necessário comportamento de consumer quase em tempo real;
- quando quiser evitar polling agressivo em tabela;
- quando o fluxo for realmente orientado a eventos.

Para cenários simples de aula, a fila por tabela continua excelente didaticamente.

## 9. Extensões práticas (modelos adicionais)

> Objetivo: mostrar rapidamente como o Oracle também pode ser usado além do relacional puro.

### 9.1. Time Series (prática simples)

Criar tabela particionada por tempo:

```sql
CREATE TABLE mod1_metrics_ts (
  ts    TIMESTAMP NOT NULL,
  host  VARCHAR2(40) NOT NULL,
  cpu   NUMBER(5,2),
  mem   NUMBER(5,2)
)
PARTITION BY RANGE (ts) (
  PARTITION p2026m01 VALUES LESS THAN (TIMESTAMP '2026-02-01 00:00:00'),
  PARTITION p2026m02 VALUES LESS THAN (TIMESTAMP '2026-03-01 00:00:00'),
  PARTITION pmax     VALUES LESS THAN (MAXVALUE)
);
```

Inserir dados:

```sql
INSERT INTO mod1_metrics_ts VALUES (TIMESTAMP '2026-01-10 10:00:00', 'srv-01', 45.2, 71.0);
INSERT INTO mod1_metrics_ts VALUES (TIMESTAMP '2026-01-10 10:05:00', 'srv-01', 52.0, 73.1);
INSERT INTO mod1_metrics_ts VALUES (TIMESTAMP '2026-02-03 09:00:00', 'srv-02', 35.7, 60.3);
COMMIT;
```

Consulta por janela de tempo:

```sql
SELECT ts, host, cpu, mem
FROM mod1_metrics_ts
WHERE ts >= TIMESTAMP '2026-01-01 00:00:00'
  AND ts <  TIMESTAMP '2026-02-01 00:00:00'
ORDER BY ts;
```

Agregação por hora:

```sql
SELECT TRUNC(ts, 'HH24') AS hora,
       AVG(cpu) AS cpu_media,
       AVG(mem) AS mem_media
FROM mod1_metrics_ts
GROUP BY TRUNC(ts, 'HH24')
ORDER BY hora;
```

### 9.2. XML (legado e integração)

Criar tabela com `XMLTYPE`:

```sql
CREATE TABLE mod1_xml_docs (
  id      NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  payload XMLTYPE
);
```

Inserir XML:

```sql
INSERT INTO mod1_xml_docs (payload)
VALUES (XMLTYPE('<pedido><id>9001</id><cliente>Ana</cliente><total>120.50</total></pedido>'));
COMMIT;
```

Consultar campos XML:

```sql
SELECT id,
       EXTRACTVALUE(payload, '/pedido/id')      AS pedido_id,
       EXTRACTVALUE(payload, '/pedido/cliente') AS cliente,
       EXTRACTVALUE(payload, '/pedido/total')   AS total
FROM mod1_xml_docs;
```
