# Revisão do Módulo 0

> Resumo curto da primeira aula: conexão Oracle, `SID`, `Service Name`, `CDB`, `PDB` e scripts SQL essenciais para começar.

## Regra mental da aula

```txt
SID = instância
Service Name = PDB
CDB = container principal
PDB = banco lógico de trabalho
```

No laboratório:

```txt
SID: FREE
Service Name: FREEPDB1
CDB: FREE
PDB: FREEPDB1
```

## Regra prática de conexão

- usar `Service Name` para laboratório, aplicação e tabelas;
- usar `SID` quando o contexto for mais administrativo;
- em Oracle, normalmente não existe `USE database`;
- a conexão entra diretamente na `PDB` correta.

## Checklist de conexão

Exemplo de conexão por `Service Name`:

```txt
Host: localhost
Port: 1522
Service Name: FREEPDB1
User: system
Role: Normal
```

Exemplo de conexão por `SID`:

```txt
Host: localhost
Port: 1522
SID: FREE
User: system
Role: Normal
```

## Queries de validação do ambiente

### Ver a instância

```sql
SELECT instance_name,
       host_name,
       status
FROM v$instance;
```

### Ver o banco principal

```sql
SELECT name AS database_name,
       open_mode,
       cdb
FROM v$database;
```

### Ver o container atual

```sql
SELECT SYS_CONTEXT('USERENV', 'CON_NAME') AS current_container
FROM dual;
```

### Ver as PDBs

```sql
SELECT con_id AS pdb_id,
       name AS pdb_name,
       open_mode AS status
FROM v$pdbs
ORDER BY con_id;
```

### Ver os services

```sql
SELECT name AS service_name
FROM v$services
ORDER BY name;
```

## Scripts essenciais de laboratório

### Criar usuário

```sql
CREATE USER jeff 
IDENTIFIED BY StrongPass123
DEFAULT TABLESPACE USERS
TEMPORARY TABLESPACE TEMP
QUOTA UNLIMITED ON USERS;
```

### Conceder privilégios

```sql
GRANT CREATE SESSION, CREATE TABLE, CREATE VIEW TO jeff;
```

### Conceder mais privilégios
```sql
GRANT CREATE SEQUENCE, CREATE PROCEDURE TO jeff;
```

### Remover usuário

```sql
DROP USER jeff CASCADE;
```

### Conceder privilégios

```sql
GRANT CONNECT, RESOURCE TO jeff;
```

> Em laboratório, `DBA` pode aparecer como atalho, mas não deve ser o padrão de aplicação.

### Criar tabela

```sql
CREATE TABLE clients (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR2(100),
    email VARCHAR2(150),
    created_at DATE DEFAULT SYSDATE
);
```

### Inserir dados

```sql
INSERT INTO clients (name, email)
VALUES ('Jefferson', 'jeff@email.com');

INSERT INTO clients (name, email)
VALUES ('Arthur', 'arthur@email.com');

COMMIT;
```

### Consultar dados

```sql
SELECT * FROM clients;
```

```sql
SELECT id, name, email
FROM clients
ORDER BY id DESC;
```

### Atualizar e remover

```sql
UPDATE clients
SET email = 'novo@email.com'
WHERE id = 1;

DELETE FROM clients
WHERE id = 2;

COMMIT;
```

### Ver tabelas do schema atual

```sql
SELECT table_name
FROM user_tables
ORDER BY table_name;
```

## Resumo final

```txt
1. Subir Oracle
2. Conectar na PDB correta
3. Entender SID e Service Name
4. Criar usuário
5. Criar tabela
6. Inserir, consultar, atualizar e remover
```
